# LICENSE Strategy Field Description

## Purpose of this file

This document describes the key decision fields used in the licensing note for the repository.

It is not a machine schema, but a maintenance-oriented field description for reviewers, users, and later project consolidation.

## Field list

1. **`repository_root_license`**  
   **Field type:** String  
   **Field description:** Primary license applied at repository root level.

2. **`repository_root_license_purpose`**  
   **Field type:** String  
   **Field description:** Short explanation of why the root license is used.

3. **`current_scope`**  
   **Field type:** String  
   **Field description:** Current practical scope of the present licensing setup.

4. **`future_split_option`**  
   **Field type:** String  
   **Field description:** Optional future plan to separate code licensing from documentation and figure licensing.

5. **`code_license_candidate`**  
   **Field type:** String  
   **Field description:** License intended or preferred for code-like material.

6. **`documentation_license_candidate`**  
   **Field type:** String  
   **Field description:** Possible later license for documentation and explanatory text.

7. **`figure_license_handling`**  
   **Field type:** String  
   **Field description:** Intended strategy for handling figure-specific reuse notes if finer control becomes necessary.

8. **`journal_agreement_distinction`**  
   **Field type:** String  
   **Field description:** Reminder that repository licensing and journal publication agreements are separate legal layers.

9. **`current_recommendation_status`**  
   **Field type:** String  
   **Field description:** Summary of the currently recommended licensing state for the project.

10. **`maintenance_note`**  
    **Field type:** String  
    **Field description:** Guidance for later repository refinement if the project grows in complexity.
