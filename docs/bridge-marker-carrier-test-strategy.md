# Bridge Marker–Carrier Test Strategy

## Purpose of this document

This document defines an internal test strategy for distinguishing **marker behavior** from **carrier behavior** in the current bridge candidate of the **Quantum–Spacetime Bridge** project.

The bridge candidate currently localizes most strongly in **weighted relational structure**. The key unresolved question is whether that layer is:

- primarily a **marker** of deeper bridge-relevant organization  
or
- already a **carrier** of bridge-relevant organization in a constitutive sense

This document turns that conceptual distinction into a practical test program.

## Core test question

The central question is:

> What happens to the bridge-relevant readable effect when the weighted relational layer is selectively altered while coarser structural summaries are held as stable as reasonably possible?

This question is stronger than asking whether the effect merely exists. It asks what structural role the weighted relational layer actually plays.

## Working hypothesis space

The current test strategy is organized around two hypothesis classes.

### H(M): Marker hypothesis

The weighted relational layer is the strongest current **visibility layer**, but not the main operative carrier of the bridge.

Under this hypothesis:
- perturbing the weighted relational structure may weaken readability
- but the deeper bridge-relevant organization should remain at least partly recoverable
- coarser or alternative representations may retain some structural trace
- the current layer is primarily diagnostic, not constitutive

### H(C): Carrier hypothesis

The weighted relational layer is not only where the bridge is best seen, but where a significant part of the bridge-relevant organization is actually borne.

Under this hypothesis:
- targeted perturbation of the weighted relational structure should directly weaken or destroy the bridge-relevant readable effect
- this collapse should be stronger than what would be expected from coarse topological or distance-geometric changes alone
- the weighted relational layer is therefore closer to an operative medium than to a mere marker surface

## Current evidential starting point

The test strategy begins from the current established internal position:

- pair-based H3 framework established
- fixed logic in place
- mixed-like sensitivity passed
- D1/D2 divergence shown
- effect does not primarily sit in topology
- effect does not primarily sit in derived distance geometry
- effect currently localizes most strongly in weighted relational structure

This means the project is already beyond pure discovery mode. The next step is role discrimination.

## General test design principle

The strategy should follow this principle:

> selectively disturb weighted relational organization while preserving as much as possible of the coarser structural shell

This means that useful perturbations are not arbitrary noise injections. They should be designed so that:

- topology is preserved where possible
- derived geometry is preserved where possible
- node and edge counts are preserved where possible
- only the weighted relational pattern is specifically degraded, flattened, scrambled, or rebalanced

The cleaner this separation is, the more meaningful the marker–carrier inference becomes.

## Main intervention families

The following intervention families are the most relevant.

### 1. Weight scrambling under stable topology

**Design idea:**  
Preserve the graph connectivity structure as much as possible, but scramble or permute the weights.

**Variants:**
- global weight permutation
- within-shell permutation
- within-neighborhood permutation
- constrained permutation preserving marginal weight distribution

**Interpretive purpose:**  
Test whether the readable bridge effect depends on the particular arrangement of weights rather than merely on topology or overall weight statistics.

**Expected pattern:**
- marker: readability may blur, but deeper signal should not fully collapse
- carrier: readability should collapse more sharply if the actual weighted arrangement matters constitutively

### 2. Weight flattening / homogenization

**Design idea:**  
Reduce variation among weights while keeping the underlying adjacency pattern.

**Variants:**
- replace weights by shell means
- replace weights by global mean
- clip extreme values
- shrink dynamic range toward uniformity

**Interpretive purpose:**  
Test whether bridge-relevant information depends on contrast and heterogeneity in the weighted relational layer.

**Expected pattern:**
- marker: signal may weaken, but core distinction may remain partially recoverable
- carrier: targeted homogenization may directly erase the effect

### 3. Local bridge-path lesioning

**Design idea:**  
Identify local high-informative weighted corridors, bridge-like edge sets, or local weighted support structures, then selectively weaken or remove them.

**Variants:**
- lesion top-k highest influence edges
- lesion local weighted clusters
- lesion candidate “support-side” neighborhoods
- lesion only structurally asymmetric weighted zones

**Interpretive purpose:**  
Test whether the bridge effect depends on specific weighted substructures rather than on diffuse global background.

**Expected pattern:**
- marker: effect should redistribute or remain partially recoverable
- carrier: local collapse should be sharper if these zones are actually bearing the bridge-relevant organization

### 4. Distance-preserving reweighting stress

**Design idea:**  
Construct perturbations that preserve derived distance-like summaries as far as possible while changing the underlying weighted relational arrangement.

**Interpretive purpose:**  
This is one of the strongest discrimination designs because it tests whether the effect survives once geometry-like surrogates are held comparatively stable.

**Expected pattern:**
- marker: if geometry already retains the deeper signal, the effect should remain substantially recoverable
- carrier: if the effect depends on weighted relations beyond geometry, it should collapse despite approximate distance preservation

### 5. Topology-preserving local asymmetry distortion

**Design idea:**  
Keep edges and adjacency structure but deliberately distort local weight balance or directional support asymmetry.

**Interpretive purpose:**  
Test whether the bridge candidate depends on fine relational organization rather than merely on graph presence.

**Expected pattern:**
- marker: coarse signal should remain if asymmetry is only a visible expression
- carrier: asymmetry manipulation may hit the bridge candidate directly

### 6. Controlled null-style reweighting series

**Design idea:**  
Generate a family of perturbations spanning from minimally altered weighted structure to progressively scrambled structure.

**Interpretive purpose:**  
Instead of a single intervention, observe the degradation curve of the bridge-relevant signal.

**Expected pattern:**
- marker: more gradual and representation-flexible degradation
- carrier: sharper threshold-like or structurally specific breakdown

## Reconstruction tests

In addition to destructive tests, the strategy should include reconstruction tests.

### Key reconstruction question

> If weighted relational structure is degraded, can the bridge-relevant effect be recovered from topology and derived distance geometry alone?

This is crucial.

If the answer is:
- **yes, substantially** → stronger marker interpretation
- **no, not without the weighted relational layer** → stronger carrier interpretation

### Reconstruction test variants

- reconstruct from pure adjacency
- reconstruct from derived distance matrix
- reconstruct from topology plus global weight distribution without local assignment
- reconstruct from coarse graph metrics only

The purpose is not to get exact recovery, but to test whether the essential bridge-readable structure survives reduction.

## Readout logic

The strategy needs clear readouts.

### Primary readout class

Readouts should target the current bridge-relevant signal layer, including:
- readability retention
- discrimination retention
- weighted relational contrast stability
- D1/D2 separation persistence
- support-side structural sensitivity

### Secondary readout class

Secondary readouts should include:
- topological stability measures
- geometry-like surrogate stability
- local perturbation scope
- global versus local collapse pattern

The point is not just to see whether something changed, but **where the loss of explanatory power occurs**.

## Expected evidence patterns

### Pattern leaning marker

A result leans marker if:
- weighted perturbation weakens readability
- but the core effect remains substantially reconstructible from coarser or alternative layers
- collapse is broad but not specifically tied to weighted relational destruction
- distance-geometric or topological surrogates still preserve significant explanatory value

### Pattern leaning carrier

A result leans carrier if:
- specifically weighted-relational perturbation causes sharp effect degradation
- topology remains broadly stable
- distance-geometric surrogates remain partly stable
- but the bridge-readable effect nonetheless collapses
- recovery from reduced surrogates fails or becomes decisively poor

### Pattern requiring caution

A result should remain undecided if:
- all representations collapse together
- perturbation is too destructive to isolate layer role
- topology, geometry, and weighting are all altered simultaneously
- marker and carrier predictions are not meaningfully separable

## Negative controls

Good negative controls are essential.

### Control class 1: topology-only stress

Apply perturbations that alter topology more strongly than weighting and observe whether the effect responds in the same way.

### Control class 2: geometry-only stress

Where possible, alter derived distance-like surrogates while leaving weighted relation assignment more stable.

### Control class 3: random low-impact perturbations

Apply structurally small, weakly targeted perturbations to check whether observed collapse is specific or merely generic fragility.

### Control class 4: permutation controls

Use randomization families that preserve certain marginals to separate structural role from trivial distribution effects.

## Decision logic

The decision logic should remain graded, not binary.

### Suggested internal decision labels

- **marker-supported**
- **carrier-leaning**
- **carrier-supported**
- **undecided**
- **test-not-informative**

This avoids forcing a yes/no claim before the evidence architecture warrants it.

## Recommended execution order

A sensible execution order is:

1. minimally invasive weight scrambling  
2. weight flattening / homogenization  
3. local lesion tests  
4. distance-preserving reweighting attempts  
5. reconstruction tests  
6. comparative control stress tests

This order moves from simpler interventions to stronger discriminators.

## What this strategy is trying to avoid

The strategy is designed to avoid several bad inferential habits:

- treating any perturbation sensitivity as carrier evidence
- treating any residual survival as marker proof
- using interventions that are too destructive to interpret
- conflating topological loss with weighted-relational loss
- making mechanistic claims from diagnostic readouts alone

## Bottom line

The marker–carrier question cannot be answered by rhetoric or by literature comparison alone.

It requires a structured intervention program in which the **weighted relational layer** is selectively stressed while coarser structural shells are held as stable as possible. Only then can the project begin to decide whether the current bridge candidate is merely the best visible readout surface or already the operative carrier of bridge-relevant organization.
