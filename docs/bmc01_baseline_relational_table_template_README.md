# BMC-01 Baseline Relational Table Template

## Purpose of this file

This file defines a **minimal baseline CSV template** for the first bridge experiment block:

- `BMC-01`
- `Weighted Relational Scramble Probe`

Its purpose is to give the experiment implementation a small, explicit, and immediately runnable baseline input.

The file is a **template and dry-run starter**, not a canonical scientific dataset. It is meant to test the BMC-01 implementation pipeline and make the required structure concrete.

## File name

`bmc01_baseline_relational_table_template.csv`

## Minimal required columns

The current BMC-01 implementation requires at least:

1. `pair_id`
2. `endpoint_a`
3. `endpoint_b`
4. `weight`

These four fields are sufficient for the variant:

- `global_weight_permutation`

## Optional but recommended columns

To support the more informative constrained variants, the template already includes:

5. `local_group`
6. `shell_label`

These fields are needed for:

- `within_local_group_weight_permutation`
- `within_shell_weight_permutation`

## Why this template already includes both optional fields

Although the absolute minimum would only need four columns, it is more practical to include both optional grouping fields from the beginning, because:

- the same file can be used for all current BMC-01 variants
- local and shell-constrained scrambling can be tested immediately
- the baseline becomes more structurally informative
- later real baseline extraction can follow the same schema

## Interpretation of the template rows

The rows represent a small weighted relational system with:

- pair identifiers
- endpoint labels
- numeric weights
- local grouping labels
- shell grouping labels

The values are synthetic and chosen only to provide:

- variation in weights
- multiple local groups
- multiple shell labels
- enough structure to make permutations nontrivial

## Minimal dry-run usage example

After placing the CSV into the repository, a first dry run could look like:

```bash
python scripts/bmc01_weighted_relational_scramble.py   --input data/bmc01/bmc01_baseline_relational_table_template.csv   --output-dir runs/BMC-01/BMC01_dryrun_001   --variant within_shell_weight_permutation   --strength medium   --seed 123
```

## Recommended placement in the repository

For a clean first dry run, a practical target would be:

```text
data/bmc01/bmc01_baseline_relational_table_template.csv
```

This keeps the file clearly separate from later real extracted baseline states.

## Field list

1. `pair_id` — String — unique identifier of the weighted pair or edge  
2. `endpoint_a` — String — identifier of the first endpoint  
3. `endpoint_b` — String — identifier of the second endpoint  
4. `weight` — Float — weighted relational value assigned to the pair  
5. `local_group` — String — grouping label for local constrained scrambling  
6. `shell_label` — String — grouping label for shell-constrained scrambling

## Notes for later real baseline extraction

When the template is replaced by a real project-derived baseline, the following should remain true:

- `pair_id` stays unique
- `endpoint_a` and `endpoint_b` remain stable identifiers
- `weight` remains numeric and directly interpretable
- `local_group` and `shell_label` should be generated by explicit, documented logic
- extraction should be reproducible from the upstream source state

## Bottom line

This template is the smallest useful baseline input for BMC-01 that already supports:

- a transparent dry run
- all current scramble variants
- later replacement by a real extracted baseline with the same schema
