# PRE_READABLE_TYPE_B_FIRST_SEGMENT_CANDIDATE_NOTE

## First concrete segment-candidate class for the pre-readable Type-B pass

**Date:** 2026-04-07  
**Status:** internal segment-candidate note

---

## 1. Purpose
This note names the first concrete segment-candidate class for the pre-readable Type-B pass.

Its purpose is to answer the next narrowing question:

> **If the selection rule is already fixed, what is the first real kind of internal Type-B subset that should now be considered as the leading candidate for the pre-readable first pass?**

This note does not yet declare one final segment as selected.
It only fixes the first plausible segment-candidate class.

This is a segment-candidate note, not a result note.

---

## 2. Why a segment-candidate class is needed
The pre-readable block now already has:

- a first organization-side marker:
  - **A1 = structured support coherence**
- a readability-side boundary:
  - **`grid_deviation_score`**
- a threshold rule
- a comparison rule
- an execution plan
- an internal Type-B transitional window binding
- and a segment/subset selection rule

What is still missing is the first concrete answer to:

- what kind of internal Type-B subset should actually be looked at first?

Without this, the selection rule remains methodologically clear but practically underdetermined.

So the first segment-candidate class is now needed.

---

## 3. Proposed first segment-candidate class
The best current first segment-candidate class is:

> **a bridge-adjacent mesoscopic transition subset inside Type-B**

A shorter working name is:

> **Type-B bridge-adjacent transition subset**

This is the proposed first concrete segment-candidate class.

---

## 4. What this candidate class means
The intended meaning of this class is:

- not global Type-B
- not fully readability-side formed region
- not weak background
- but a mesoscopic internal subset lying close to emerging bridge-support organization while still below clear readability lock-in

This is the current best first real candidate class for the pre-readable pass.

---

## 5. Why this class should come first
This class is the best first choice because it sits exactly between the two failure zones.

### Too early / too weak failure zone
- generic background
- weak overlap-dominated zones
- regions where A1 likely cannot become meaningfully active

### Too late / too formed failure zone
- clearly readability-side formed zones
- regions where `grid_deviation_score` is likely already strongly above the readability threshold
- regions where the pre-readable question is already lost

The bridge-adjacent mesoscopic transition subset lies between these extremes.

That makes it the correct first segment-candidate class.

---

## 6. Why “bridge-adjacent” matters
The class is not just “transitional” in a vague sense.
It is specifically:
- **bridge-adjacent**

That matters because the pre-readable question should be asked where bridge-relevant structure is plausibly beginning to gather.

A transition zone far from emerging bridge-support organization would be a weaker first target because:

- A1 may not become active enough
- the whole pass may drift toward weak structure or background noise

So the first candidate class must remain close to the bridge-support side of Type-B.

---

## 7. Why “mesoscopic” matters
The class is also specifically:
- **mesoscopic**

That matters because:

- a single tiny local point may be too fragile or too noisy
- a global full-family zone is too mixed and too broad

A mesoscopic subset is preferable because it is:

- small enough to interpret
- large enough to carry real structure
- narrow enough not to blur multiple regimes together

This makes it the correct scale for the first candidate class.

---

## 8. Why “transition subset” matters
The class must remain:
- **transitional**

That means:
- support-side organization is already more than background
- but readability-side articulation is still not clearly locked in

This is essential because the pre-readable pass is specifically a timing-layer test.

If the subset is not transitional, it does not really test:
- whether organization can appear before readability

So the first candidate class must remain explicitly transitional.

---

## 9. Positive profile of the first segment-candidate class
The bridge-adjacent mesoscopic transition subset should ideally have the following profile:

- some internal structure already more than weak background
- A1 plausibly capable of becoming materially active
- local or mesoscopic bridge-support relevance
- readability-side articulation still plausibly below threshold
- not yet globally formed
- interpretable enough for disciplined review

This is the target profile.

---

## 10. What this candidate class excludes
The class is useful partly because of what it excludes.

### Excluded class 1 — Global Type-B cases
Too broad, mixes too many internal regimes.

### Excluded class 2 — Fully formed readability-side subsets
Too late, boundary likely already crossed.

### Excluded class 3 — Bare weak background subsets
Too early, A1 likely too weak.

### Excluded class 4 — Highly anomalous exotic subsets
Too hard to calibrate as the first pass.

So the proposed class is the first clean middle ground.

---

## 11. Why this class is better than “the strongest structured segment”
It might be tempting to simply choose the strongest-looking structured region.
That would be a mistake.

Why?
Because the strongest structured region may already be:
- clearly readability-side formed

That would turn the first pre-readable pass into a trivial failure.

The first candidate class is therefore **not**:
- the strongest structure

It is:
- the earliest bridge-adjacent mesoscopic subset where structure is already nontrivial but not yet clearly readability-side formed

That distinction matters.

---

## 12. Relationship to A1
This class is chosen because it gives A1 the fairest first habitat.

A1 needs:
- more than weak background
- but not already readability-dominated terrain

The bridge-adjacent mesoscopic transition subset is currently the best class meeting those conditions.

So this class is the most plausible first substrate for testing whether A1 can function as an organization-side marker.

---

## 13. Relationship to the readability boundary
This class is also chosen because it gives the readability boundary a meaningful job.

The first candidate class must be one in which:
- `grid_deviation_score` could still plausibly remain below threshold

If the class were already too readability-side formed, then the boundary test would be trivial and the pre-readable comparison would teach almost nothing.

So the class is deliberately chosen to preserve nontrivial boundary uncertainty.

- more than background
- less than fully formed readability-side structure
- bridge-relevant
- and narrow enough to support a disciplined first-pass review

That is the current first segment-candidate standard for the pre-readable Type-B block.
