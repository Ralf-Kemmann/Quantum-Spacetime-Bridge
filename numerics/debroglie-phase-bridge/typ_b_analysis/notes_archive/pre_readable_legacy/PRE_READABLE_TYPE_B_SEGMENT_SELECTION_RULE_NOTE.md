# PRE_READABLE_TYPE_B_SEGMENT_SELECTION_RULE_NOTE

## Segment / subset selection rule for the internal Type-B transitional window

**Date:** 2026-04-07  
**Status:** internal selection-rule note

---

## 1. Purpose
This note translates the already defined internal Type-B transitional bridge-adjacency window into a more concrete segment / subset selection rule.

Its purpose is to answer the next narrowing question:

> **How should one actually select the first internal Type-B segment or subset that is most suitable for the pre-readable first pass?**

This note does not yet select one final segment by name.
It defines the rule by which such a segment should now be chosen.

This is a selection-rule note, not a result note.

---

## 2. Why a selection rule is needed
The pre-readable block now already has:

- a first organization-side marker:
  - **A1 = structured support coherence**
- a readability-side boundary:
  - **`grid_deviation_score`**
- a readability threshold rule
- a first comparison rule
- a first case window
- and a binding note tying that window to an internal Type-B transitional bridge-adjacency zone

What is still missing is the next practical narrowing step:

- how to extract or identify the actual internal segment

Without that step, the transition window remains conceptually well placed, but still operationally too loose.

So the segment-selection rule is now required.

---

## 3. Core selection logic
The segment should be chosen where all of the following are simultaneously plausible:

1. **organization is already nontrivial**
2. **readability has not yet clearly locked in**
3. **the zone is internal to Type-B rather than external background**
4. **the zone is transitional rather than final-state-like**
5. **the zone is interpretable enough for a first-pass case review**

This is the core selection logic.

---

## 4. The two-sided selection principle
The selection rule should always be read as a two-sided filter.

### Lower-side filter
Reject zones where organization is still too weak, too noisy, or too close to generic overlap/background.

### Upper-side filter
Reject zones where readability-side articulation is already too strong and the case is already effectively formed.

The desired segment lies between these two failures.

A short version is:

> **Choose the first zone that is already more than background, but not yet clearly formed.**

That is the key principle.

---

## 5. Positive inclusion criteria
A candidate Type-B segment should be considered eligible for first-pass use only if most of the following are true.

### 5.1 Internal support-side organization is already visible
There should be some reason to think that A1 could become materially active here.

This means:
- not mere background
- not flat weak overlap
- not obviously structureless

### 5.2 The zone is bridge-adjacent
The segment should lie near emerging bridge-support structure rather than in unrelated or remote low-structure territory.

### 5.3 Readability-side articulation is still plausibly below threshold
`grid_deviation_score` should still have a realistic chance to remain on the weak side of the readability boundary.

### 5.4 The segment is not already globally formed
The local or mesoscopic structure should still feel transitional rather than fully locked into readability-side form.

### 5.5 The segment can be described as a coherent subset
The selected segment should be concrete enough to review as:
- one subset
- one local window
- or one interpretable internal slice

These are the positive inclusion criteria.

---

## 6. Exclusion criteria
A segment should be excluded from first-pass use if one or more of the following is clearly true.

### 6.1 Too close to background
The zone shows too little nontrivial organization for A1 to be meaningful.

### 6.2 Too close to already readable form
The zone already shows strong readability-side articulation and therefore no longer tests the pre-readable distinction.

### 6.3 Too global
The zone is so broad that it mixes:
- weak background
- transitional structure
- and clearly formed regions

This makes the first pass muddy.

### 6.4 Too exceptional
The zone is dominated by idiosyncratic behavior so strong that it stops being a good first calibration target.

### 6.5 Too hard to describe reproducibly
If the segment cannot be named or delimited in a reproducible way, it should not be the first-pass target.

These are the exclusion criteria.

---

## 7. Preferred segment shape
For the first pre-readable pass, the preferred segment shape should be:

- **local or mesoscopic**
- **internally coherent enough to describe**
- **narrow enough not to blur multiple regimes together**
- **wide enough that A1 and the readability boundary are both meaningful**

This means the first segment should not be:
- a single isolated point unless unavoidable
and not:
- a global full-family sweep

A short phrase is:

> **small enough to interpret, large enough to carry structure**

That is the preferred shape.

---

## 8. Recommended selection rule in words
The recommended selection rule should be:

> **Choose the first internal Type-B segment for which support-side organization is already stronger than background and plausibly bridge-adjacent, while readability-side articulation remains still below the formed-readability threshold. Reject segments that are either too weakly organized or already clearly readability-side formed.**

This is the preferred full verbal rule.

A shorter version is:

> **Select the earliest bridge-adjacent Type-B subset that is already structured but not yet clearly readable.**

That is the operational core.

---

## 9. Practical subset-selection sequence
A reasonable first practical sequence is:

### Step 1
Identify candidate internal Type-B subsets that are not merely background.

### Step 2
Remove subsets that are already obviously readability-side formed.

### Step 3
From the remaining subsets, prefer the one that is:
- closest to emerging bridge-support organization
- but still plausibly below readability-threshold crossing

### Step 4
Choose the narrowest interpretable subset satisfying that profile.

This is the recommended first selection sequence.

---

## 10. Relationship to A1
The rule is built so that A1 has a fair chance to become informative.

The chosen segment should be one where:
- A1 could reasonably become materially active
without
- being forced to rely on already-readable form

So the selection rule is partly a habitat rule for A1.

That is why the lower-side exclusion is so important:
- too little organization → A1 says nothing
- too much readability → A1 is no longer pre-readable-side meaningful

---

## 11. Relationship to the readability boundary
The rule also protects the readability boundary from becoming trivial.

If the selected segment is too late:
- `grid_deviation_score` is likely already on the crossed side

Then the pre-readable question is already lost.

So the rule deliberately searches for the first zone in which readability still has a real chance to remain:
- **weak**

This is why the upper-side exclusion matters just as much as the lower-side one.

---

## 12. Preferred first-pass interpretation states
The selection rule implicitly defines three useful practical states.

### State A — Too early / too weak
Reject:
- insufficient organization-side structure

### State B — Transitional and eligible
Keep:
- organization plausibly active
- readability plausibly still weak

### State C — Too late / too formed
Reject:
- readability-side articulation already clearly strong

The first-pass target is State B only.

---

## 13. Safe summary sentence
A useful internal sentence is:

> The first pre-readable segment should be chosen as the earliest internal Type-B bridge-adjacent subset that is already more than background but not yet clearly readability-side formed.

A shorter version is:

> **Neither background nor formed structure: choose the bridge-adjacent transition subset in between.**

Both are good steering lines.

---

## 14. Bottom line
The concrete segment / subset selection rule for the first pre-readable Type-B window is:

> **Select a local or mesoscopic internal Type-B bridge-adjacent subset where organization is already nontrivial and plausibly stronger than background, while readability-side articulation remains still below the formed-readability threshold. Exclude subsets that are either too weakly organized or already clearly formed.**

That is the current selection-rule standard for the first pre-readable Type-B segment.
