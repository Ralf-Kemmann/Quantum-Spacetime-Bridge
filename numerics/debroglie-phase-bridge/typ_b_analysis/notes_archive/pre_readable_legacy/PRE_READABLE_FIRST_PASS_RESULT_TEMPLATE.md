# PRE_READABLE_FIRST_PASS_RESULT_TEMPLATE

## Template for documenting the result of a pre-readable first-pass comparison

**Date:** 2026-04-07  
**Status:** internal result template

---

## 1. Purpose
This template is used to document the outcome of the first pre-readable comparison block.

Its purpose is to prevent three common failures:

- forgetting why a case was or was not counted as pre-readable
- rescuing weak cases later through vague narrative reinterpretation
- losing the exact relation between organization-side activity and readability-side boundary status

This is a result template, not a synthesis note.

It should be used after:
- Candidate Marker A1 has been evaluated for a case
- the readability-side boundary has been evaluated for the same case
- and the comparison rule has been applied

## 5. First-pass comparison outcome
Now apply the actual pre-readable comparison rule.

### Rule reminder
A case may count as plausibly pre-readable only if:
- **A1 = materially active**
and
- **readability boundary = weak**

### Allowed case outcomes
Assign exactly one:

- **Plausible pre-readable case**
- **Not pre-readable**
- **Ambiguous / borderline case**

### Template
> First-pass case outcome: **[outcome]** because [short reason based on A1 status and readability-boundary status].

---

## 6. No-rescue block
This section is mandatory.

Its purpose is to record what is **not** being claimed for this case.

### Required prompts
- Is the case being over-read from weak A1 alone?
- Is weak readability being mistaken for pre-readable organization without sufficient A1 activity?
- Is the case already too readability-side formed to count as pre-readable?
- Is overlap-only presence being mistaken for organization?

### Required statement
> This case should not be promoted beyond its assigned first-pass status without explicit new justification.

Keep this sentence.

---

## 7. Promotion decision
State clearly what happens next for this case.

### Allowed promotion outcomes
- no promotion
- keep as exploratory-only case
- carry forward as plausible pre-readable candidate case
- revisit only after marker refinement
- revisit only after readability-boundary refinement

### Template
> Promotion decision: **[state the outcome]** because [short reason].

---

## 8. Next-step block
The case review must end with one concrete next-step statement only.

### Allowed next-step types
- stop for now
- refine A1 definition
- refine readability threshold
- compare this case against another case
- keep as current candidate pre-readable case
- re-evaluate after marker/boundary refinement

### Template
> Next step: **[one concrete next step only]**

Do not list many possible next steps.
Force one.

---

## 9. Minimal artifact reminder
This result template should ideally be stored together with:
- A1 marker definition
- readability boundary definition
- readability threshold rule
- the applied comparison rule
- and any raw case notes used for judgment

The result note should not float free from its basis.

---

## 10. What this template forbids
This template forbids:

- calling a case pre-readable because readability is weak while A1 is weak too
- calling a case pre-readable because it is conceptually attractive
- promoting a case even though the readability boundary is already crossed
- forgetting whether overlap-only explanations were excluded
- blurring “ambiguous” into “basically positive”

This is a discipline template.

---

## 11. Safe summary sentence
A useful one-line summary at the end is:

> The first pre-readable pass is complete for this case, and the case status has been recorded without rescue logic.

A second acceptable version is:

> First-pass pre-readable case outcome recorded: organization/readability comparison complete.

Either sentence is acceptable.

---

## 12. Bottom line
A pre-readable first-pass case review should leave behind a disciplined result record that states:

- how A1 behaved
- how the readability boundary was classified
- whether the case qualifies as plausibly pre-readable, not pre-readable, or ambiguous
- what is not being claimed
- and what the single next step is

This template defines that result format.
