# PRE_READABLE_TYPE_B_ELIGIBLE_CANDIDATE_SUBSET_LIST_TEMPLATE

## Template for the eligible candidate subset list in the first pre-readable Type-B pass

**Date:** 2026-04-07  
**Status:** internal candidate-list template

---

## 1. Purpose
This template is used to record the **eligible candidate subset list** after the field of view has been fixed and the no-go zones have been excluded.

Its purpose is to answer one practical question:

> **Which internal Type-B subsets remain genuinely eligible for first-pass pre-readable ranking once the hard exclusions have already been applied?**

This is a candidate-list template, not a result note.

---

## 2. Why this template is needed
The pre-readable Type-B block now already has:

- a field of view
- explicit no-go zones
- a bridge-adjacent mesoscopic transition target class
- a subset-selection workflow
- and a selection-record template

What is still needed in practice is the intermediate list between:

- **excluded terrain**
and
- **ranked candidates**

That intermediate list is:

- the **eligible candidate subset list**

Without this list, selection can drift too quickly from map-level filtering to one chosen subset.

This template prevents that jump.

---

## 3. What counts as an eligible candidate subset
A subset should appear on this list only if it has already survived the no-go filter.

That means it is:

- not weak background
- not already readability-side formed
- not an anomalous oddity
- not an overmixed regime zone

and still plausibly satisfies the positive target profile:

- bridge-adjacent
- mesoscopic
- more than background
- not yet clearly readability-side locked in
- interpretable enough for later review

Only such subsets belong on this list.

## 6. Minimal inclusion rule
A subset should be included on the eligible list only if the answer to all of the following is at least plausibly “yes”:

1. Is it more than weak background?
2. Is it not already clearly readability-side formed?
3. Is it not too anomalous for first-pass calibration?
4. Is it not too regime-mixed?
5. Is it bridge-adjacent?
6. Is it mesoscopic enough to interpret?
7. Could A1 plausibly become active here?
8. Could the readability boundary still plausibly remain weak here?

If any of these clearly fails, the subset should not appear on the eligible list.

|---|---|---|---|---|---|---|

---

## 8. What this list must not contain
This list must not contain:

- zones already excluded as no-go
- vague “interesting regions” with no reproducible boundary
- subsets included merely because they fit the desired story
- fully formed readability-side zones kept as secret backups
- giant broad regions that still need heavy internal filtering

This list is for **eligible** candidates only.

---

## 9. What happens after this list
The eligible candidate subset list is not the end of selection.

The next step is:

> **rank the eligible subsets by pre-readable suitability and choose the narrowest interpretable winning subset**

So this list is the bridge between:
- exclusion
and
- ranking

That is its role.

---

## 10. Safe summary sentence
A useful internal sentence is:

> The eligible candidate subset list should contain only those internal Type-B subsets that are already more than background, not yet clearly readability-side formed, and still interpretable enough to be ranked as first-pass pre-readable candidates.

A shorter version is:

> **Only bridge-adjacent mesoscopic transition subsets that survived the no-go filter belong on the eligible list.**

Both are good steering lines.

---

## 11. Bottom line
The eligible candidate subset list is the first positive shortlist inside the pre-readable Type-B selection workflow.

It should contain only subsets that:

- survived the no-go filter
- are bridge-adjacent
- are mesoscopic
- are more than background
- and may still remain below the readability-side threshold

This template defines how that shortlist should be recorded.
