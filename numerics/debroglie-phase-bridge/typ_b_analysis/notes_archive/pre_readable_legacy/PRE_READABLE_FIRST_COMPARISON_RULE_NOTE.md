# PRE_READABLE_FIRST_COMPARISON_RULE_NOTE

## First comparison rule between A1 and the readability boundary

**Date:** 2026-04-07  
**Status:** internal comparison-rule note

---

## 1. Purpose
This note defines the first explicit comparison rule for the pre-readable-organization block.

Its purpose is to answer one narrow operational question:

> **Under what condition should a case count as a plausible pre-readable case when comparing Candidate Marker A1 against the readability-side boundary?**

This note is needed because the pre-readable block now already has:

- one first organization-side marker:
  - **A1 = structured support coherence**
- one first readability-side boundary:
  - **`grid_deviation_score`**

What was still missing was the explicit rule connecting the two.

This is a comparison-rule note, not a result note.

---

## 2. Why this rule is needed
Without an explicit comparison rule, the pre-readable block remains vulnerable to two opposite mistakes:

### Mistake 1 — Overclaiming
Any case with weak organization-side signal could be narrated as “pre-readable” merely because readability is also weak.

### Mistake 2 — Under-distinguishing
A case with genuinely nontrivial early organization could be ignored because no formal comparison to the readability boundary exists.

So the block now needs one simple rule:

- when does A1 count as already active enough,
- while readability remains still weak enough,
- that the case deserves pre-readable status?

This note provides that rule.

---

## 3. Core comparison logic
The intended comparison logic is:

> **A case should count as plausibly pre-readable only when the organization-side marker A1 is already clearly nontrivial, while the readability-side boundary marker remains still weak enough that the case cannot yet be treated as readability-side formed.**

This is the central rule in words.

A shorter version is:

> **Pre-readable = A1 already active, readability boundary not yet crossed.**

That is the basic logic.

---

## 4. The two required sides of the rule
The rule has two parts, and both are mandatory.

## 4.1 Organization-side condition
A1 must be:
- clearly nontrivial,
- not merely noise-level,
- and not reducible to overlap-only presence

If A1 is weak, near-noise, or only overlap-like, then the case should not be promoted as pre-readable.

---

## 4.2 Readability-side condition
The readability-side boundary marker must remain:
- weak,
- incomplete,
- below threshold,
- or otherwise not yet strong enough to count as readable-form articulation

If the readability-side boundary is already strong, the case is no longer pre-readable.

Both sides must be satisfied together.

---

## 5. First-pass verbal rule
The first-pass rule should be stated as follows:

> **Assign a case as plausibly pre-readable only if A1 is already materially active as an organization-side marker while the readability boundary marker remains below the readable-articulation threshold. Otherwise do not assign pre-readable status.**

This is the preferred full verbal rule.

A shorter version is:

> **A1 active + readability boundary weak = plausible pre-readable case.**

---

## 6. What “A1 materially active” should mean
For the comparison rule, A1 should count as materially active only if:

- its value is clearly above minimal-noise / minimal-hint level,
- it is not merely a tiny fluctuation,
- and it is interpretable as early structured support organization rather than raw overlap dressed up

So “A1 active” should be read conservatively.

It should not mean:
- any positive signal at all

It should mean:
- enough early organization-side activity to deserve real attention.

---

## 7. What “readability boundary weak” should mean
For the comparison rule, the readability-side boundary should count as weak only if:

- the chosen readability-side marker is still below the readable-form threshold,
- or remains too weak / too incomplete to count as clearly formed readable articulation

So “boundary weak” should mean:
- not yet readable enough to count as formed

This should also be read conservatively.

It should not mean:
- merely “not maximal”
or
- “not globally obvious”

It should mean:
- still below the threshold for readable-form presence.

---

## 8. Recommended first binary rule
The cleanest first-pass binary rule is:

### Assign **plausible pre-readable case**
if:
- **A1 = active**
and
- **readability boundary = weak**

### Assign **not pre-readable**
otherwise

This is the simplest correct first comparison rule.

---

## 9. Optional future refinement
A richer future classification may later distinguish:

- plausible pre-readable
- ambiguous / borderline
- already readability-side formed
- weak / no organization case

But for the first pass, the cleanest start is the binary comparison rule above.

This keeps the block disciplined.

---

## 10. What the rule must not do
The first comparison rule must **not** do any of the following:

### 10.1 Accept weak A1 as enough
If A1 is barely active, the case should not be promoted merely because readability is also weak.

### 10.2 Accept strong readability as still pre-readable
If the readability boundary is already crossed, the case is not pre-readable anymore, even if A1 is strong.

### 10.3 Treat overlap-only as organization
If A1 is not distinguishable from overlap-only presence, the rule should not count the case as pre-readable.

### 10.4 Drift into ontology
This rule should not be used to claim anything broad about appearance, geometry, or ontology.
It is only a narrow comparison rule.

These restrictions are essential.

---

## 11. Recommended current interpretation states
The comparison rule naturally implies three practical states, even if the first pass uses only binary assignment.

### State A — Plausible pre-readable
- A1 active
- readability boundary weak

### State B — Already readability-side formed
- A1 active
- readability boundary strong

### State C — No useful early-organization signal yet
- A1 weak
- readability boundary weak or strong

For the very first pass, only State A should receive the positive pre-readable label.

---

## 12. Safe summary sentence
A useful internal sentence is:

> A case should count as pre-readable only when early organization-side coherence is already materially active while the readability-side articulation marker is still below the formed-readability threshold.

A shorter version is:

> **Pre-readable means: organization already there, readability not yet.**

Both are good steering lines.

---

## 13. Bottom line
The first pre-readable comparison rule between A1 and the readability boundary is:

> **Assign plausible pre-readable status only when A1 is materially active as an organization-side marker and the readability-side boundary remains below the readable-articulation threshold. Otherwise do not assign pre-readable status.**

This is the current first explicit comparison rule for the pre-readable block.
