# PRE_READABLE_READABILITY_BOUNDARY_NOTE

## First readability-side boundary for the pre-readable-organization block

**Date:** 2026-04-07  
**Status:** internal boundary note

---

## 1. Purpose
This note defines the first readability-side boundary for the pre-readable-organization block.

Its purpose is to answer one narrow but necessary question:

> **What counts, in the current project, as already readable enough that a case should no longer be treated as pre-readable?**

This boundary is needed because the pre-readable block can only work if it has:

- one organization-side candidate marker
and
- one readability-side boundary

Without the second, “pre-readable” remains too soft.

This is a boundary note, not a result note.

---

## 2. Why the readability boundary must be explicit
The pre-readable block is built on the distinction between:

- deeper organization
and
- later readable articulation

That means the block requires not only a candidate marker for early organization, but also a clear upper threshold for:

- readable enough
- formed enough
- articulated enough

Without such a boundary, several problems appear:

- later readable structure can be mislabeled as “pre-readable”
- weak early organization can be overclaimed because no readability threshold exists
- and the distinction between organization and articulation becomes rhetorical rather than operational

So the readability boundary is mandatory.

---

## 3. Intended role of the readability-side boundary
The readability-side boundary is not a proof target.
It is a limit.

Its role is to mark the region where structure has become:

- visibly articulated enough
- formed enough
- organized enough in readable terms

that the case should no longer be treated as merely pre-readable.

So the readability boundary functions as:

- an upper articulation boundary
- a later-form boundary
- and an anti-inflation boundary

That is its correct role.

---

## 4. Working conceptual definition
The current working definition should be:

> **Readability begins where structure is no longer merely deeper organized, but has become articulated enough to count as formed in a readable bridge-relevant sense.**

A shorter version is:

> **Pre-readable ends where structure becomes clearly formed enough to read.**

That is the intended distinction.

---

## 5. Recommended first readability-side boundary marker
For the current project state, the most natural first readability-side boundary marker is:

- **`grid_deviation_score`**

This is the preferred boundary marker because the current internal reading already places it most plausibly on the:

- morphology
- form
- readable organization side

rather than on the missing selector side or the earliest organization side.

So for the pre-readable block, `grid_deviation_score` is the cleanest first articulation-side ceiling.

---

## 6. What the readability boundary is allowed to contain
A readability-side boundary marker may contain:

- later form articulation
- morphology-like organization
- readable structure strength
- later bridge-shape regularity or deviation information
- signals that are naturally stronger once structure is visibly formed

These are acceptable boundary-side ingredients.

---

## 7. What the readability boundary must not be confused with
The readability boundary must not be confused with:

### 7.1 Early organization itself
It is not the candidate marker.
It is the upper articulation limit that early organization must remain below to count as pre-readable.

### 7.2 Compatibility selection
It is not the missing selector layer.
It is later than that.

### 7.3 Broad ontology or full appearance
The readability boundary should remain narrower than:
- full appearance
- geometry-wide claims
- or broad ontological articulation

It is a first practical readability boundary, not a full philosophy of appearance.

---

## 8. What it means for a case to remain below the readability boundary
A case remains plausibly pre-readable only if:

- the organization-side marker is already nontrivial
- but the readability-side boundary marker remains weak, incomplete, unstable, or below the readable-articulation threshold

This is the core comparison logic.

In other words:

- organization-side activity may already be there
- but later readable articulation must not yet be strongly present

That is the pre-readable zone.

---

## 9. What it means for a case to cross the readability boundary
A case should be treated as having crossed the readability boundary if one or more of the following holds:

### 9.1 Form articulation is already clearly present
The structure is no longer only implied; it is visibly organized enough to count as formed.

### 9.2 Readability-side marker is already strong
The chosen readability-side marker is no longer weak or borderline, but clearly active.

### 9.3 The case is already easy to read as formed bridge structure
A human or operational reader no longer needs to infer organization from subtle markers because readable articulation is already obvious.

Once this happens, the case should not be called pre-readable.

---

## 10. Relationship to Candidate Marker A1
Candidate Marker A1 asks:
- is structured support coherence already active as early organization?

The readability boundary asks:
- has the case already become too articulated to count as pre-readable?

So the real pre-readable test will later be:

> **Is A1 already clearly active while the readability boundary marker remains weak enough that the case is not yet fully readable?**

This is the key comparison.

---

## 11. Why `grid_deviation_score` is a good first boundary marker
`grid_deviation_score` is a good first readability-side boundary because it already has a plausible current role as:

- form-side
- morphology-side
- readable articulation-side

This makes it much better suited here than:

- raw overlap
- rigidity-side support persistence
- or broad ontological wording

It is late enough to function as readability-side boundary, but still specific enough to stay operational.

---

## 12. What the readability boundary must not do
The readability boundary must not become:

- a full bridge-quality score
- a general theory-success score
- a broad ontological criterion
- or a moving target redefined after seeing a case

It should remain:

- narrow
- explicit
- later than organization markers
- and fixed before comparison begins

This is essential.

---

## 13. Required discipline for the pre-readable block
For the first pre-readable comparison frame, the readability-side boundary must be:

- fixed before case comparison begins
- explicit and shared across cases
- identical for the tested organization-side marker comparisons
- and not redefined mid-pass to rescue a desired interpretation

This is mandatory.

---

## 14. Safe summary sentence
A useful internal sentence is:

> The first readability-side boundary should be the point at which later form articulation becomes strong enough that a case is no longer merely organized beneath readability, but already readable as formed bridge-relevant structure.

A shorter version is:

> **Pre-readable ends where form becomes clearly readable enough to count as formed.**

Both are good steering lines.

---

## 15. Bottom line
The first readability-side boundary for the pre-readable block should currently be defined using:

- **`grid_deviation_score` as the preferred first articulation-side boundary marker**

Its role is to mark the point above which a case is no longer plausibly pre-readable.

So the future comparison logic is:

- **A1 active**
- **readability boundary still weak**
= plausible pre-readable case

Whereas:

- **A1 active**
- **readability boundary already strong**
= no longer pre-readable, already readability-side formed

That is the current first readability-boundary standard for the pre-readable block.
