# COMPATIBILITY_FIRST_FOCAL_UNIT_AND_NEIGHBORHOOD_FIXATION_NOTE

## Fixation note for the first focal unit and immediate neighborhood in the compatibility first pass

**Date:** 2026-04-07  
**Status:** internal fixation note

---

## 1. Purpose
This note fixes the one remaining narrowing step in the first compatibility local context:

> **the focal local support unit**
and
> **the immediate bridge-adjacent neighborhood boundary**

Its purpose is to answer one practical question:

> **How do we turn the first local-context record from a structurally correct but still slightly generic launch candidate into a concrete enough context for full first-pass acceptance?**

This is a fixation note, not a result note.

---

## 2. Why this fixation step is needed
The first local-context record instance was already judged:

- structurally correct
- meaningful for A1
- meaningful for B1
- jointly frame-compatible with overlap, rigidity, and DLBSR-class

But it was still classified as:

- **Borderline context — refine once**

The reason was narrow and explicit:
- the focal unit was still too generic
- the immediate neighborhood boundary was still too generic

So this note now performs the one allowed refinement step.

---

## 3. Core fixation principle
The fixation should obey one central principle:

> **Choose the smallest local support kernel that is already more than background and can carry both coherence and perturbation meaning, then define its neighborhood as the immediate bridge-adjacent support shell only.**

A shorter version is:

> **One focal kernel, one immediate shell, nothing wider.**

That is the governing principle.

---

## 4. Focal-unit fixation
The first focal unit should now be fixed as:

> **one single local bridge-adjacent support relation or one minimal local support kernel that is already more than weak background and lies inside the internal Type-B support-relevant terrain**

This is the fixed focal-unit standard.

### What this means
The focal unit is:

- not a broad regime
- not a mesoscopic cluster
- not a diffuse local patch
- not a global support region

It is:
- the smallest local support-bearing unit that can still support both an A1 reading and a B1 perturbation reading

This is the correct first focal-unit fixation.

---

## 5. Why this focal unit is right for A1
This focal unit is suitable for A1 because it provides:

- one local center of support organization
- one immediate neighborhood around which agreement and conflict can be read
- and a minimal local anchor from which coherence can be evaluated without already broadening into a form-scale object

So the focal unit is:
- small enough to stay early
- but real enough to carry support structure

That is why it is correct for A1.

---

## 6. Why this focal unit is right for B1
This focal unit is suitable for B1 because it provides:

- one clear perturbation target
- one minimally interpretable support-bearing element
- and one local object whose weakening/removal can still be read as early fragility rather than as regime-scale destruction

So the focal unit is:
- small enough to perturb cleanly
- but real enough that its perturbation can still matter

That is why it is correct for B1.

---

## 7. Immediate-neighborhood fixation
The immediate neighborhood should now be fixed as:

> **the focal unit plus only its directly adjacent bridge-adjacent local support relations inside the same internal Type-B support-relevant region**

This is the fixed neighborhood standard.

### What this means
The neighborhood is:

- not a second-shell neighborhood
- not a broad mesoscopic regime patch
- not a mixed transition corridor
- and not a global support neighborhood

It is:
- one immediate first shell around the focal unit only

This is the correct first neighborhood fixation.

---

## 8. Why this neighborhood is right for A1
This immediate neighborhood is suitable for A1 because it is the smallest shell in which:

- agreement can be read as more than focal-unit self-description
- conflict can be read as more than isolated noise
- and local support organization can still be evaluated without broadening into later readability-side form

So the neighborhood is:
- wide enough for coherence
- narrow enough to remain pre-stabilization

That is why it is correct for A1.

---

## 9. Why this neighborhood is right for B1
This immediate neighborhood is suitable for B1 because it is the smallest shell in which:

- local perturbation response can still propagate meaningfully
- local support loss can be read as more than pointwise disappearance
- and fragility can still be interpreted as local dependence rather than as broad regime collapse

So the neighborhood is:
- wide enough for perturbation response
- narrow enough to remain early and local

That is why it is correct for B1.

---

## 10. What the fixation explicitly excludes
This fixation step explicitly excludes:

### Exclusion A — Broadening the focal unit
The focal unit must not be widened into:
- a local cluster
- a mini-regime
- or an already mesoscopic support object

### Exclusion B — Broadening the neighborhood
The neighborhood must not be widened into:
- second-order shells
- regime-mixed transition bands
- or readability-near broad patches

### Exclusion C — Adaptive reshaping
The focal unit and neighborhood must not be re-expanded later just because the first reading is inconvenient.

These exclusions are mandatory.
